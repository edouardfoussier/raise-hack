# Diffender

**Ship the demo, not just the code.** — [diffender.com](https://diffender.com)

Diffender is an **AI-native design-system reviewer** *and* a **deterministic demo-video generator**, delivered as a Cursor MCP server + CLI. One engine, two jobs:

1. **Design-drift review** — on any code change, Diffender replays your *real* running UI **before vs after**, and a vision model reasons **in your design tokens** to classify the change as an `intentional_redesign`, `accidental_regression`, or `platform_constraint` — then **drafts the fix**. Opens a self-contained before/after report.
2. **Demo video** — describe a flow (or let the AI plan it), and Diffender **drives your real app deterministically**, filming a captioned, branded, share-ready demo. Real backend, real data — not a flaky screen recording.

Both share one deterministic Playwright engine: **plan once → replay identically** on before *and* after, so comparisons are honest and demos are reproducible.

> Built for the **RAISE Summit Hackathon 2026 · Cursor track**. The 60-second submission video you're watching was **generated by Diffender itself.**

## Why it's different

- **Reasons in design tokens, proposes the fix** — not a pixel diff (Percy/Chromatic) and not a Figma-only linter. It bridges the real running UI *and* the token system.
- **Deterministic** — Playwright DOM replay, identical every run. Cursor's agent "proof-of-work" clips are non-deterministic computer-use recordings; ours are reproducible and show *before + after*.
- **Drives a live backend** — mobile emulation, synthetic touch + iOS keyboard, and verified write-mode against a real dev database (a delivery reception increments a real stock counter, confirmed via the API).

## How it works

```
git (HEAD vs working)  →  Playwright replay (real DOM, synthetic cursor/captions)  →
  VLM verdict in tokens (+ proposed fix)  →  self-contained HTML report
                                          ↘  compose intro/outro + Gradium voice-over  →  demo video
```

- **Review**: `render.ts` (before/after screenshots) → `vlm.ts` (`getVerdict`) → `report.ts`.
- **Demo**: `planner.ts` (`agentFlow` plans steps from a goal) → `flow.ts` (`recordLiveFlow` with captions) → `compose.ts` (branded intro/outro) → `gradium.ts` (voice-over).

## Quickstart

```bash
cd mcp-server && npm install            # pulls Chromium
echo "ANTHROPIC_API_KEY=sk-..." > .env  # or OPENROUTER_API_KEY
npm run review                          # static drift review on the sample app
npx tsx src/cli-demo.ts                 # goal + URL → captioned deterministic demo video
```

Cursor commands: `/drift` (static), `/drift-motion` (interaction), `/drift-flow` (user-flow video). See [`docs/STATE.md`](docs/STATE.md) and [`docs/VISION.md`](docs/VISION.md).

## 🧠 The engine (`mcp-server/`, Node+TS via `tsx`; Playwright + ffmpeg)
- `cli-demo.ts` — **generic generator**: `DEMO_URL` + `DEMO_GOAL` → `agentFlow` (plan) → `recordLiveFlow` (captioned replay) → `composeVideo` → mp4. Env knobs: `DEMO_DEVICE` (mobile→iPhone13+touch+iOS keyboard / desktop→cursor), `DEMO_HEADERS` (JSON extraHTTPHeaders, e.g. Vercel bypass), `DEMO_INIT` (JS initScript, e.g. seed auth token), `DEMO_READONLY`, `DEMO_VOICE`/`DEMO_VOICE_ID`, `DEMO_SCRIPT` (edited per-step VO lines). **Audio-driven sync**: one narration line per step, each step dwells for its TTS duration → voice matches action, **length adaptive**. Captions render in a **bottom subtitle band** (compose), never covering the app.
- `cli-repo.ts` — repo → serve (static or `npm run dev`) → same pipeline → video.
- `cli-submission.ts` + `gradium.ts` — the 60s submission video (Edouard's cloned Gradium voice; voice id in env `GRADIUM_VOICE_ID`, default hard-coded). Script in the file's `SCRIPT[]`.
- `cli-script.ts` — LLM writes per-step narration lines sized to a duration.
- `moneyshot.ts` + `report.ts` + `vlm.ts` — the **review money-shot** (Statement One centerpiece): introduce a drift → `render`/`git` → `getVerdict` (classification+reasoning+proposed_diff, 97%) → self-contained flame report PNG. Re-run reverts the drift.
- `planner.ts` (`agentFlow`) + `providers.ts` (`selectPlannerModel`) — **NVIDIA Nemotron via Nebius Token Factory** plans the flow (see `NEMOTRON.md`); vision review stays on Claude.
- `flow.ts` (`recordLiveFlow`: device, pointer touch, keyboard, extraHTTPHeaders, captions→timed `CaptionCue[]`), `compose.ts` (intro/outro cards + subtitle band, flame `#FF5A1F`), `caption-overlay.js`, `tap-overlay.js`, `keyboard-overlay.js`, `cli-deja-*.ts` (deja-bu: stock, reception via seeded session, checklist, keyboard).
